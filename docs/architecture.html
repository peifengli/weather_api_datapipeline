Scalability Paths

  • 10× frequency (near-real-time): Replace EventBridge poll → Kinesis Data Streams; Lambda consumes stream → triggers Glue micro-batch or writes directly to Firehose → S3.
  • Multi-region / global cities: Extend city list to any OWM-supported location; Glue Catalog federation across regions; partitioned by region= prefix.
  • Snowflake migration: Snowpipe auto-ingest on S3 ObjectCreated events; swap Athena for Snowflake SQL; DBT profiles unchanged (DuckDB → Snowflake connector).
  • Dashboard scale-out: ECS Service Auto Scaling (target-tracking on ALB RequestCountPerTarget); add ElastiCache for DuckDB query result caching.
  • Real-time push: OpenWeatherMap Enterprise WebSocket push → Kinesis Firehose → S3 → trigger Lambda for immediate processing (eliminates polling latency entirely).

Bad raw data from OWM Schema validation before processing; failed records quarantined to s3://.../quarantine/ for inspection
process_weather re-run Idempotent by design — overwrites same hive partition; no duplicates regardless of how many times it runs
ECS task crash ALB health check (/_stcore/health × 3 failures) → ECS replaces task automatically; CloudWatch logs last output
S3 read failure in dashboard @st.cache_data(ttl=300) serves stale data for up to 5 min; UI shows last-updated timestamp so users know
Terraform state conflict Remote state in S3 with DynamoDB locking; CD pipeline concurrency: cancel-in-progress: false prevents concurrent applies

Why ECS Fargate + ALB over App Runner?

  • WebSocket support: App Runner's ingress layer strips/mishandles the HTTP Upgrade: websocket header — the handshake fails silently and the server returns HTML instead of a 101 response. ALB natively forwards WebSocket upgrade headers (RFC 6455), which Streamlit requires for its bidirectional _stcore/stream connection.
  • Sticky sessions: ALB lb_cookie stickiness ensures each browser's WebSocket and subsequent HTTP requests always reach the same ECS task — critical when running multiple replicas.
  • Idle timeout control: ALB idle_timeout = 3600 keeps long-lived WebSocket connections alive. App Runner has a fixed 120 s inactivity timeout.
  • Cost at single-instance scale: ~$25/month (ALB ~$16 + Fargate ~$9) vs App Runner ~$30+/month for equivalent memory.
